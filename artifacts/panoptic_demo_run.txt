TEMPORAL COLOUR CONSISTENCY
---------------------------
Thing instances are matched frame-to-frame by mask IoU within the same semantic class
(--iou-match-threshold, default 0.3); a match reuses the previous stable track id and
therefore the previous colour, otherwise a new id is minted. Stuff classes (road, sky,
vegetation, ...) skip matching entirely — their colour is a pure function of the class id,
so it is identical in every frame and across runs.

Evidence: "car #16" (dark red) and "car #6" (purple) hold the same ids and the same
colours from frame 39 through frame 78 — 39 frames apart.


NOTES / LIMITATIONS
-------------------
  * Instance numbers climb over the clip (car #125 by the end) because distant cars enter
    and leave frame, and a track lost for even one frame is re-minted. Colour stability
    between *adjacent* frames — what the overlay actually needs — is unaffected.
  * Labels are suppressed for segments below --min-area (default 1500 px) so tiny specks
    do not clutter the frame; those regions are still coloured.
  * Confidence is not drawn: Mask2Former's per-segment score is not meaningful enough to
    display alongside a class name.
